01 · Giới thiệu

Gemini CLI
AI Agent trong Terminal

Gemini CLI là open-source AI agent mang sức mạnh của Gemini trực tiếp vào terminal của bạn. Không chỉ là chatbot — đây là một agent có khả năng đọc/ghi file, chạy lệnh shell, tìm kiếm web, và kết nối với bất kỳ MCP server nào.

02 · Cài đặt

⚙️ Cài đặt Gemini CLI

Yêu cầu: Node.js 18+. Có 3 cách cài:

1
NPX — Không cần cài (nhanh nhất)
Dùng ngay không cần cài đặt, phù hợp để thử nghiệm lần đầu.
bash 
npx @google/gemini-cli
2
NPM Global — Cài vĩnh viễn (khuyến nghị)
Dùng lệnh gemini từ bất cứ đâu sau khi cài.
bash 
npm install -g @google/gemini-cli

# Sau đó chạy:
gemini
3
Conda (môi trường độc lập)
Dành cho người muốn cô lập môi trường Node.js.
bash 
conda create -y -n gemini_env -c conda-forge nodejs
conda activate gemini_env
npm install -g @google/gemini-cli
⚠️ Lưu ý Node.js version
Gemini CLI yêu cầu Node.js 18 trở lên. Chạy node --version để kiểm tra. Nếu cần upgrade: nvm install 20 && nvm use 20

04 · Chế độ dùng

💬 Các chế độ sử dụng

Gemini CLI có 3 chế độ chính, mỗi chế độ phù hợp với tình huống khác nhau:

Interactive Shell
Inline Prompt
Pipe Input
💬 Interactive Shell — Chế độ chính
Gõ gemini để vào interactive shell. Bạn có thể chat nhiều lượt, agent sẽ nhớ context của conversation. Đây là chế độ phổ biến nhất để làm việc với codebase.
gemini — interactive shell
$ gemini
Welcome to Gemini CLI v0.21.0
✓ Authenticated as your-email@gmail.com (Gemini 2.5 Pro)
❯ Explain the architecture of this codebase
◆ Thinking... reading project structure
⚡ read_file(package.json)
⚡ glob("src/**/*.ts")
✦ This is a Spring Boot microservice (Rider Service)...
⚡ Inline Prompt — One-shot
Dùng flag -p để chạy một prompt không cần vào interactive mode. Rất hữu ích cho scripting và automation.
bash 
# Prompt đơn giản
gemini -p "Explain the architecture of this codebase"

# Output dạng JSON (cho scripting)
gemini -p "List all API endpoints" --output-format json

# Stream JSON (cho realtime events)
gemini -p "Run tests and deploy" --output-format stream-json
📥 Pipe Input — Xử lý stdin
Pipe nội dung file/output vào Gemini CLI để xử lý. Kết hợp mạnh với các CLI tool khác.
bash 
# Phân tích log file
cat error.log | gemini -p "What caused this error?"

# Review git diff
git diff | gemini -p "Review this code change"

# Dịch README
cat README.md | gemini -p "Translate this to Vietnamese"
05 · Slash Commands

⌨️ Slash Commands

Trong interactive mode, gõ / để xem tất cả commands. Đây là các lệnh quan trọng nhất:

/help
Xem tất cả commands và keyboard shortcuts. Điểm khởi đầu tốt nhất.
/chat save <name>
Lưu conversation hiện tại. Dùng sau để /chat resume.
/chat resume <name>
Tiếp tục conversation đã lưu trước đó.
/chat share
Export conversation thành Markdown file để chia sẻ.
/clear
Xóa màn hình terminal, giữ nguyên context conversation.
/reset
Xóa toàn bộ context conversation, bắt đầu lại từ đầu.
/memory show
Xem toàn bộ context đã được load (GEMINI.md + memory files).
/memory add <text>
Thêm memory mới vào GEMINI.md của dự án.
/settings
Mở settings panel: model, approval mode, theme, plugins...
/tools
Xem danh sách tất cả tools đang available (built-in + MCP).
/extensions
Quản lý extensions: list, install, update, remove.
/bug
Report bug trực tiếp lên GitHub issues của Gemini CLI.

06 · Built-in Tools

🔧 Built-in Tools

Gemini CLI có sẵn các tools mạnh mẽ mà agent sẽ tự chọn và dùng khi cần. Bạn không cần gọi thủ công.

📁
File Operations
read_file, write_file, edit_file, glob — đọc/ghi/sửa file trên local.
🔍
Search & Grep
grep_search, glob — tìm kiếm pattern trong codebase, regex search nhanh.
💻
Shell Commands
run_shell_command — chạy bất kỳ lệnh terminal nào (npm, git, mvn, kubectl...)
🌐
Web Fetch
fetch_url — lấy nội dung trang web, parse HTML, đọc documentation online.
🔎
Google Search
google_search — tìm kiếm real-time trên Google, grounding response với thông tin mới nhất.
🖼️
Multimodal Input
Nhận diện và xử lý ảnh, PDF làm input. Generate app từ wireframe ảnh.
💡 Agent tự quyết định dùng tool nào
Bạn chỉ cần mô tả task bằng ngôn ngữ tự nhiên. Agent sẽ tự quyết định cần dùng tool nào và sẽ hỏi xác nhận trước khi thực hiện hành động có thể gây ra thay đổi (ghi file, chạy lệnh).

🛡️ Approval Modes

Kiểm soát mức độ tự động của agent. Dùng Shift+Tab để chuyển giữa các mode:

ModeMô tảPhù hợp
Default (Manual) Hỏi xác nhận trước mỗi tool call có tác động Mọi tình huống — an toàn nhất
Auto-Edit Tự động file edits, hỏi khi chạy shell Refactoring code quen thuộc
YOLO (Full Auto) ⚠️ Tự động mọi thứ, không hỏi Sandbox, automation pipeline
Plan Mode new Read-only: chỉ đọc, lập kế hoạch, không ghi file Research, planning trước khi implement
07 · MCP Servers

🔌 MCP — Model Context Protocol

MCP cho phép Gemini CLI kết nối với các external service. Đây là cách mở rộng khả năng của agent vượt ra ngoài local file system.

✅ Cách kết nối MCP server
Thêm vào file ~/.gemini/settings.json (global) hoặc .gemini/settings.json trong project (local).
json — ~/.gemini/settings.json 
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://user:pass@localhost/mydb"]
    },
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": { "BRAVE_API_KEY": "BSA..." }
    }
  }
}

📦 MCP Servers phổ biến

🐙 GitHub @modelcontextprotocol/server-github
🗄️ PostgreSQL @modelcontextprotocol/server-postgres
🗃️ SQLite @modelcontextprotocol/server-sqlite
☁️ Cloud Run (Google) Extension từ Gemini CLI Gallery
🔍 Brave Search @modelcontextprotocol/server-brave-search
📦 Filesystem @modelcontextprotocol/server-filesystem
💡 Với Spring Boot / PostgreSQL project
Kết nối MCP server PostgreSQL để agent có thể query schema trực tiếp: "Describe the rider_forms table" → agent sẽ tự query DB để trả lời.
08 · GEMINI.md

📄 GEMINI.md — Project Context

File GEMINI.md được Gemini CLI tự động load mỗi khi bạn bắt đầu session trong thư mục dự án. Đây là cách cung cấp persistent context cho agent.

📍 Vị trí tìm kiếm
Gemini CLI tìm GEMINI.md theo thứ tự: thư mục hiện tại → thư mục cha (recursive) → home directory. Bạn có thể có nhiều file nested cho nhiều level context.
markdown — GEMINI.md (ví dụ Spring Boot project) 
# Project Context

## Stack
- Java 17 + Spring Boot 3.2
- PostgreSQL 14 + Liquibase
- Kafka (consumer)
- Maven build

## Architecture
- Hexagonal Architecture (Ports & Adapters)
- DDD aggregates
- Package: com.mbageas.rider

## Key Conventions
- Service layer: @Transactional on Application Services only
- Entity IDs: UUID
- Exception: RiderException(code, message)

## Common Commands
# Build: mvn clean install -DskipTests
# Test:  mvn test
# Run local: mvn spring-boot:run -Dspring.profiles.active=local

📝 Tips viết GEMINI.md hiệu quả

  • Mô tả stack và version cụ thể — tránh agent suggest wrong syntax
  • Liệt kê architecture patterns bạn đang dùng (DDD, Hexagonal, CQRS...)
  • Ghi các convention quan trọng (naming, exception handling, transaction boundary)
  • Thêm common commands thường dùng (build, test, run)
  • Đề cập các file/folder quan trọng để agent navigate nhanh hơn
  • Ghi chú các gotcha / anti-patterns cần tránh trong project
09 · Headless Mode

🤖 Headless / Non-interactive Mode

Dùng Gemini CLI trong script, CI/CD pipeline, hoặc automation — không cần user input.

bash 
# CI: auto code review khi có PR
gemini -p "Review the staged changes, find bugs and security issues" \
  --output-format json \
  --yolo \
  --allowed-tools read_file,grep_search

# Generate changelog từ git log
git log --oneline -20 | gemini -p "Generate a changelog from these commits"

# Custom slash command trong headless mode
gemini "/joke Chuck Norris"

# Giới hạn tools được phép dùng (bảo mật)
gemini -p "Analyze this codebase" \
  --allowed-tools read_file,glob,grep_search \
  --yolo
🔒
--allowed-tools
Whitelist các tool được phép dùng. An toàn hơn cho automation pipeline.
📤
--output-format json
Output dạng JSON để parse trong script. Kết hợp với jq để xử lý.
10 · Plan Mode

🗺️ Plan Mode — Research-first

Plan mode là chế độ read-only, giúp agent nghiên cứu và lập kế hoạch mà không có nguy cơ thay đổi code.

✅ Khi nào dùng Plan Mode
Trước khi implement feature lớn, migrate database, hay refactor codebase phức tạp — dùng Plan Mode để agent phân tích toàn bộ rồi đề xuất implementation plan.
gemini — plan mode
❯ Plan how to migrate RiderForm to use the new PersonProfile schema
◆ [PLAN MODE] Research only, no file changes
⚡ read_file(src/domain/RiderForm.java)
⚡ grep_search("PersonProfile", "src/**")
⚡ glob("src/infrastructure/persistence/**")
✦ Migration Plan:
1. Add person_profile_id FK to rider_forms table
2. Create PersonProfileRepository port
3. Update RiderFormApplicationService to resolve profile
4. Liquibase migration: V1.5.0__add_person_profile_fk.sql
Estimate: ~3 hours, 6 files
❯ Looks good, proceed with implementation
◆ Switching to Edit Mode...

⚙️ Bật Plan Mode mặc định

interactive 
# Trong interactive shell:
/settings → Default Approval Mode → Plan

# Hoặc dùng Shift+Tab để toggle giữa các mode
11 · Use Cases

💡 Use Cases thực tế

Các tình huống thực chiến phù hợp nhất với Gemini CLI, đặc biệt trong môi trường Spring Boot / Microservices:

🐛 Debug & Fix lỗi

prompt examples 
# Phân tích exception
cat stacktrace.log | gemini -p "Find root cause and suggest fix"

# Fix lỗi OptimisticLockingFailure
> Fix the ObjectOptimisticLockingFailureException in RiderFormService
  and add proper retry mechanism

# Tìm N+1 query
> Find N+1 query problems in the repository layer and suggest fixes

✨ Feature Development

prompt examples 
# Tạo Liquibase migration
> Create a Liquibase migration to add person_profiles table
  with UUID PK, name, dob, gender fields, with proper indexes

# Tạo unit test
> Write unit tests for HealthDeclarationMappingService,
  cover recursive question tree and edge cases

# Review code trước commit
git diff --staged | gemini -p "Review for bugs, security issues, and Spring best practices"

📚 Hiểu codebase nhanh

prompt examples 
> Explain the flow from Kafka consumer to database persistence
  in this Rider Service codebase

> Draw an ASCII diagram of the hexagonal architecture layers
  and list which classes belong to each layer

> What does RiderFormRequestService.processRequest() do step by step?

🔒 Pentest / Security Review

prompt examples 
> Check all API endpoints for missing authentication/authorization
> Find SQL injection vulnerabilities in repository layer
> Check if sensitive data (PII, passwords) is being logged
12 · Tips & Tricks

⚡ Tips & Tricks

✅ Prompt tốt vs Prompt kém

❌ Quá chung chung
Fix the bug in my code
✅ Cụ thể, có context
Fix the NullPointerException in RiderFormService.java line 87 when user has no profile
❌ Một lúc quá nhiều
Refactor toàn bộ service layer, viết test, thêm logging, sửa bug
✅ Từng bước rõ ràng
1. First, just show me the plan. 2. Then refactor only the service layer.
❌ Không nói format mong muốn
Explain the system
✅ Chỉ rõ output format
Explain the system as ASCII diagram with 3-sentence description for each layer

🛠️ Tips thực tế

  • Bắt đầu với Plan Mode cho task lớn — tránh agent làm sai hướng rồi mới phát hiện
  • Dùng /chat save khi context conversation đang tốt để resume sau
  • Ctrl+C ngay lập tức nếu thấy agent đang làm sai hướng — đừng để nó tiếp tục
  • Cung cấp ví dụ trong prompt: "Implement similar to how RiderFormService handles validation"
  • Review file changes trước khi confirm — dùng git diff sau mỗi batch thay đổi
  • GEMINI.md là investment — 30 phút viết GEMINI.md tốt tiết kiệm hàng giờ context trong tương lai
  • Pipe git diff vào Gemini để review PR trước khi push — nhất là với team có code review nghiêm
  • Dùng --output-format json khi cần parse output trong shell script
⚠️ Không dùng YOLO mode trong production repo
Full Auto mode (YOLO) rất tiện nhưng nguy hiểm trong codebase quan trọng. Agent có thể xóa/sửa file theo cách không ngờ tới. Luôn commit trước khi dùng YOLO, hoặc chỉ dùng trong branch tạm.
