Core Component

🎛 Agent Manager — Mission Control

Đây là điểm khác biệt lớn nhất của AntiGravity. Thay vì chatbox tuyến tính phải chờ, Agent Manager là dashboard quản lý nhiều agents song song bất đồng bộ.

🚀
Dispatch không cần chờ
Gửi task mới ngay khi agent khác đang chạy. Mỗi task có agent instance riêng xử lý độc lập — 5 bugs xử lý đồng thời.
📊
Real-time status
Xem progress, artifacts đã tạo, pending approvals, và reasoning của từng agent — tất cả trên cùng dashboard.
📦
Artifacts = bằng chứng
Agent không chỉ nói "đã fix" — nó tạo Task List, Implementation Plan, Code Diff, Walkthrough recording để bạn verify trước khi approve.
🌐
Workspaces & Playground
Workspace: project folder với file control. Playground: scratch area không có workspace, prototype nhanh, convert sau nếu cần.

📋 Cách dùng Agent Manager hiệu quả

  • Dùng @filename và @foldername để inject file context vào task ngay trong input
  • Planning mode cho complex tasks — agent tạo Task List + Implementation Plan artifact trước khi viết bất kỳ dòng code nào
  • Fast mode cho tasks đơn giản như rename, format, thêm log line — không tạo plan, execute ngay
  • Review artifact kỹ ở Implementation Plan — đây là lúc dễ phát hiện sai hướng nhất, trước khi agent dùng nhiều compute
  • Comment trực tiếp vào artifact để agent điều chỉnh — không cần bắt đầu lại từ đầu
Core Component

📝 Editor View

Editor View là môi trường coding quen thuộc — kế thừa VS Code nhưng AI được tích hợp sâu hơn bất kỳ plugin nào.

⌨️
Tab Completions
Autocomplete inline context-aware từ toàn bộ workspace, hiểu codebase của bạn không chỉ file đang mở.
💬
Inline Commands
Select code → Cmd+K để ra lệnh trực tiếp. Agent sửa tại chỗ, không cần sang chat.
🔄
Bidirectional
Bạn edit file, agent thấy ngay. Agent tạo file mới, bạn thấy ngay trong Explorer. Real-time hai chiều.
💡 Editor vs Agent Manager
Agent Manager: task phức tạp, multi-file, cần plan, cần verify qua browser. Editor View: đang hands-on code, cần gợi ý inline nhanh, hoặc chỉnh trực tiếp file agent vừa tạo.
Core Component

🌐 Browser Subagent

Tính năng độc đáo nhất của AntiGravity — agent thực sự điều khiển Chrome để test và verify, không phải giả lập.

🖱
Full browser control
Click, fill forms, scroll, navigate — agent dùng Chrome thật qua Chrome Extension. Không phải headless, không phải mô phỏng.
👁
Đọc và hiểu trang
DOM capture, screenshot, markdown parsing — agent "đọc" được trang web, hiểu content và layout thực tế.
🧪
Automated E2E testing
Agent tự deploy app, mở browser, test từng flow, chụp screenshot — tất cả tự động. Bạn review Walkthrough artifact sau.
📹
Walkthrough recordings
Agent tạo recording của toàn bộ test session — Walkthrough artifact giúp bạn verify kết quả trực quan như xem video demo.
⚠️ JavaScript Execution Policy
Browser subagent có thể chạy JS trong trang. Request Review (recommended) — hỏi trước khi chạy JS. Always Proceed — tự động nhưng có risk Prompt Injection từ malicious webpage content. Disabled — an toàn nhất nhưng hạn chế automation.
Core Concept

📦 Artifacts — Hệ thống "Bằng chứng"

Artifacts giải quyết "Trust Gap" — thay vì tin lời agent nói "đã fix", bạn có bằng chứng cụ thể để verify. Đây là UX innovation quan trọng nhất của AntiGravity.

Artifact Type Khi nào xuất hiện Nội dung Bạn cần làm gì
📋 Task List Ngay khi nhận task (Planning mode) Danh sách subtasks có structure và dependency Review, comment để điều chỉnh direction
🏗 Implementation Plan quan trọng Sau Task List, trước khi code Technical details: files cần tạo/sửa, approach, tradeoffs Review kỹ nhất đây — dễ phát hiện sai hướng
📄 Code Diff Sau khi agent viết/sửa code Diff của tất cả files thay đổi Review changes, approve hoặc request chỉnh sửa
🖼 Screenshot Sau browser test Ảnh chụp UI trước/sau, kết quả test visually Verify UI trông đúng như mong đợi
🎬 Walkthrough video-like Sau E2E testing Recording của agent test flow trên browser thật Xem như video demo — verify full flow
📊 Architecture Diagram Khi planning complex system Sơ đồ kiến trúc, data flow, component diagram Review và comment ngay trên diagram
⭐ Pro Tip
Đừng skip artifact review để "code nhanh hơn". Thời gian review Implementation Plan là thời gian tiết kiệm được từ việc undo sai hướng sau đó. Rigorous interrogation ở Artifact stage = chất lượng output cao hơn.
Customization #1

📜 Rules — System Instructions luôn chạy ngầm

Rules là hướng dẫn bất biến luôn active — agent đọc và tuân theo mọi lúc, không cần bạn nhắc lại. Khác với Workflows (cần trigger), Rules hoạt động như system prompt tự động.

🌍
Global Rules
Áp dụng cho mọi project. Triết lý coding cá nhân hoặc tổ chức.

Vị trí: ~/.gemini/GEMINI.md
📁
Workspace Rules
Chỉ cho project hiện tại, override Global Rules.

Vị trí: .agent/rules/ trong project root

📄 Ví dụ: ~/.gemini/GEMINI.md

Markdown · Global Rules 
# GEMINI.md — Global Rules (áp dụng mọi project)

## Language
- Luôn dùng TypeScript thay vì JavaScript
- Python: type hints bắt buộc (Python 3.10+)
- Không bao giờ hardcode API keys hoặc secrets

## Code Quality
- Mọi public method phải có JSDoc/docstring
- Error handling: không silent catch — luôn log hoặc rethrow
- Tên biến: descriptive, không viết tắt trừ khi rất phổ biến

## Testing
- Viết test cùng lúc với code, không phải sau
- Test file đặt cạnh source: foo.ts → foo.test.ts

## Security
- Không generate hardcoded credentials dù là example
- SQL: luôn parameterized queries, không string concat

📄 Ví dụ: .agent/rules/backend.md

Markdown · Workspace Rules 
# Backend Rules — Payment Service

## Stack (bắt buộc dùng)
- Runtime: Node.js 22 + TypeScript strict mode
- Framework: Fastify 5 (không dùng Express)
- DB: PostgreSQL với Prisma ORM
- Validation: Zod cho tất cả external input

## Architecture
# Layered architecture — không mix concerns:
- routes/       ← HTTP layer (thin, chỉ parse/validate)
- services/     ← Business logic
- repositories/ ← Database access
- domain/       ← Types, interfaces, value objects

## API Design
- Response format: { data, error, meta: { requestId, timestamp } }
- Versioning: /api/v1/ prefix
- Error: AppError class với code + statusCode + message

## Không được làm
- Không dùng any type
- Không trả về stack trace ra client trong production
- Không query không có WHERE clause và LIMIT
💡 Rules vs Workflows
Rules = passive, luôn active ngầm, giống system prompt. Workflows = active, chỉ chạy khi bạn trigger bằng /slash command. Rules là luật, Workflows là quy trình có thể gọi lại.
Customization #2

🔄 Workflows — Quy trình trigger-on-demand

Workflows là saved prompts được trigger bằng /slash command. Định nghĩa một lần, dùng mãi. Chain workflows lồng nhau để tạo pipeline phức tạp.

🌍
Global Workflows
Dùng được mọi project.

Vị trí: ~/.gemini/antigravity/global_workflows/
📁
Workspace Workflows
Chỉ trong project hiện tại.

Vị trí: .agent/workflows/ trong project root

📄 Ví dụ: .agent/workflows/code-review.md

Markdown · /review workflow 
# Workflow: review
# Trigger: /review
# Mô tả: Code review toàn diện cho staged changes

1. Chạy `git diff --staged` để lấy danh sách thay đổi
2. Đọc từng file thay đổi
3. Phân tích theo thứ tự ưu tiên:
   - 🔴 Security issues (injection, auth bypass, data leak)
   - 🔴 Correctness bugs (logic errors, null deref, off-by-one)
   - 🟡 Performance issues (N+1 queries, missing index, O(n²))
   - 🟡 Style violations (theo .agent/rules/)
   - 🟢 Suggestions (refactoring, naming, docs)
4. Tạo Review Report artifact với format:
   ## Summary
   [X critical · Y warnings · Z suggestions]
   ## Issues
   [file:line | severity | giải thích | code fix]
5. Nếu có critical issue: propose code fix ngay

📄 Ví dụ: .agent/workflows/ship-feature.md (Chain workflow)

Markdown · /ship-feature — chained workflow 
# Workflow: ship-feature
# Trigger: /ship-feature
# Mô tả: Pipeline hoàn chỉnh từ code đến PR

1. Chạy workflow /generate-tests
   → Tạo unit tests cho code vừa viết
2. Chạy: `npm test` hoặc `npx vitest run`
   → Phải PASS trước khi tiếp tục
3. Chạy workflow /review
   → Code review, không có critical issue mới proceed
4. Chạy: `git add -A && git commit -m "[auto] ..."` 
5. Tạo GitHub PR qua MCP với:
   - Title: mô tả ngắn gọn feature
   - Body: link to Task List artifact + summary of changes
   - Reviewers: lấy từ CODEOWNERS nếu có
Customization #3

⚡ Skills — Progressive Disclosure Knowledge

Skills giải quyết "tool bloat" — thay vì load hết context vào agent (chậm, tốn token, confuse), Skills chỉ được load khi request khớp với skill's description.

🔑 Nguyên lý Progressive Disclosure
Agent biết general programming từ Gemini 3. Skill cung cấp specialized knowledge của team bạn: conventions, business rules, domain expertise. Khi user hỏi "run database migration", agent scan available skills, tìm db-migration skill, load nó vào context và thực thi đúng theo team standard.
🌍
Global Skills
Dùng mọi nơi.
~/.gemini/antigravity/skills/
📁
Workspace Skills
Riêng cho project.
.agent/skills/{name}/SKILL.md

📄 Cấu trúc một Skill

.agent/skills/db-migration/
├── SKILL.md← mô tả + instructions (agent đọc khi match)
├── examples.md← ví dụ migration thực tế trong project
└── scripts/← helper scripts agent có thể gọi
└── run-migration.sh
Markdown · .agent/skills/db-migration/SKILL.md 
---
name: database-migration
description: >  
  LOAD khi user nhắc đến: "migration", "migrate database",
  "schema change", "Prisma migrate", "alter table", "add column".
---

# Skill: Database Migration

## Project DB Stack
- ORM: Prisma với PostgreSQL 16
- Migration files: prisma/migrations/
- Staging DB: postgresql://staging.internal/appdb

## Quy trình bắt buộc
1. Cập nhật prisma/schema.prisma
2. Run: `npx prisma migrate dev --name {snake_case_name}`
3. Review generated SQL trong prisma/migrations/
4. Test staging: `DATABASE_URL=$STAGING npx prisma migrate deploy`
5. Chỉ production sau khi staging OK

## Naming convention
snake_case, mô tả rõ ràng:
- add_user_email_index ✅
- migration_1 ❌

## Không được làm
- Không edit migration file sau khi đã commit
- Không xóa migration files cũ
- Không dùng raw SQL, luôn qua Prisma client
Customization #4

🔌 MCP Tools — Kết nối services bên ngoài

AntiGravity hỗ trợ Model Context Protocol (MCP) — cho phép agent gọi tools kết nối với databases, GitHub, Jira, và bất kỳ service nào có MCP server.

🐙
GitHub MCP
Đọc issues, tạo PR, xem diff. Agent tự tạo PR sau khi xong feature.
🗄️
Database MCP
Query DB trực tiếp để lấy schema, data thực tế. Agent không cần bạn paste.
📋
Jira / Linear MCP
Đọc tickets. "Fix bug từ PROJ-123" — agent đọc description tự động.
JSON · .agent/mcp.json 
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}" }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://localhost/mydb"]
    }
  }
}

Security

🛡 Agent Security

Khi agent có quyền truy cập terminal và browser, security là critical. AntiGravity có hệ thống phân quyền chi tiết.

✅
Allow List
Domains agent được phép truy cập không cần hỏi. File: ~/.gemini/antigravity/browserAllowlist.txt
🚫
Deny List
Commands agent không bao giờ được chạy — dù ở mode "Always Proceed". Ví dụ: rm -rf, DROP TABLE
💉
Prompt Injection Risk
Content từ external website có thể chứa instructions giả mạo cho agent. Dùng "Request Review" mode để có checkpoint kiểm soát.
📤
Data Exfiltration
Secure Mode ngăn agent gửi data ra ngoài domain allowlist. Dùng cho codebases chứa data nhạy cảm.
🚨 Nguyên tắc bắt buộc
Không dùng Agent-driven mode cho production codebases. Luôn dùng Review-driven hoặc Secure mode. Không cấp GITHUB_TOKEN production vào MCP — dùng token có scope giới hạn nhất có thể.

💡 Chọn model theo task
Planning mode + Gemini 3 Pro High: complex feature, architecture decisions. Gemini 3 Pro Low hoặc Sonnet: review, debug, refactor thông thường. Fast mode: rename, format, simple fixes — model nào cũng được.

Best Practices

⭐ Best Practices từ Google & Community

🏗
Đừng skip artifact review
Interrogate kỹ Implementation Plan trước khi agent code. Sai hướng phát hiện ở plan = tiết kiệm nhiều giờ undo sau này.
📝
Rules ngắn gọn, checkable
Rules quá dài loãng context. "Không dùng var" tốt hơn "viết code tốt". Mỗi rule phải có thể verify được.
⚡
Workflow cho task lặp lại
Bất kỳ task bạn làm hơn 2 lần/tuần → tạo workflow. /review, /generate-tests, /create-pr là bộ 3 cơ bản nhất.
🎯
Mô tả như kiến trúc sư
"Build auth với JWT, refresh tokens, RBAC" tốt hơn "viết code login". Bạn là architect, agent là senior engineer.
🔒
Luôn dùng Review-driven
Agent-driven tiện nhưng nguy hiểm trên codebase thật. Review-driven cho checkpoint approve trước mỗi hành động quan trọng.
🧠
Dùng @context reference
@file.ts, @folder/, @README.md — reference trực tiếp thay vì paste content. Chính xác hơn và không tốn context window.
⭐ Core Philosophy
AntiGravity không thay thế developer — nó amplify. Đầu tư vào Rules, Workflows, Skills = đầu tư vào năng suất dài hạn. Agent quality = context quality. Context tốt → Artifacts tốt → Code tốt.
