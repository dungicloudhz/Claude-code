Bắt đầu

🚀 Bắt đầu nhanh trong 3 bước

1
Tạo tài khoản tại claude.ai
Đăng ký bằng email hoặc Google. Gói Free đủ để trải nghiệm. Gói Pro ($20/tháng) tăng giới hạn tin nhắn, thêm Projects, Artifacts và ưu tiên truy cập model mới nhất. Gói Team/Enterprise dành cho tổ chức với shared Projects và admin controls.
2
Gửi tin nhắn đầu tiên
Nhấp vào ô soạn thảo ở trang chủ, gõ bất kỳ câu hỏi nào bằng ngôn ngữ tự nhiên — tiếng Việt hoặc tiếng Anh đều được. Không cần lệnh đặc biệt, không cần syntax riêng.
3
Iterate và tinh chỉnh
Đọc phản hồi. Nếu chưa đúng ý: yêu cầu chỉnh sửa, hỏi follow-up, hoặc nói rõ hơn điều bạn cần. Claude nhớ toàn bộ context trong conversation — bạn không cần giải thích lại từ đầu.

🔑 Nguyên tắc cốt lõi

  • Nói thẳng, không cần lịch sự thái quá — "Viết hàm tính tổng mảng" tốt hơn "Bạn có thể vui lòng giúp tôi viết hàm tính tổng mảng không ạ?"
  • Paste nguyên liệu thô — Code, error log, stack trace, text cần xử lý. Đừng mô tả bằng lời — show thẳng.
  • Hỏi follow-up trong cùng conversation — Claude nhớ toàn bộ context. "Giải thích thêm dòng 5" hoặc "Chỉnh sửa để handle null" đều hiểu ngay.
  • Phản hồi trực tiếp — "Câu trả lời này không đúng vì..." rõ hơn "Không đúng rồi bạn ơi".
  • Yêu cầu format cụ thể — "Chỉ code, không giải thích", "Trả lời bằng JSON", "Ngắn gọn dưới 5 dòng".

📁 Projects — Làm việc theo dự án

Projects là tính năng mạnh nhất cho developer và team. Nhóm conversations, chia sẻ context và file cho cả project.

1
Tạo Project và đặt tên mô tả
Sidebar trái → New Project. Ví dụ: "MB Ageas — Declaration Service", "Marketing Q1 2026", "Side Project — URL Shortener".
2
Viết Project Instructions (System Prompt)
Đây là hướng dẫn áp dụng cho MỌI conversation trong project. Ví dụ tốt:

Bạn là senior Java developer. Stack: Spring Boot 3.4, Java 21, PostgreSQL 16.
Project: MB Ageas Life insurance platform, microservices trên AWS EKS.
Luôn trả lời bằng tiếng Việt. Code phải có exception handling đầy đủ.
Ưu tiên performance và thread-safety. Không dùng deprecated APIs.
3
Upload tài liệu project
Thêm: ERD, API spec (Swagger/OpenAPI), README, coding guidelines, architecture diagram. Tất cả conversations trong project đều access được file này.
4
Tạo conversations trong project
Mỗi conversation là một task riêng. Tên gợi nhớ: "Fix N+1 query OrderService", "Design payment webhook handler", "Review auth middleware".
✅ Best practice thực tế
Đừng dồn mọi thứ vào 1 conversation dài. Tạo conversation mới cho mỗi feature/bug — conversation ngắn → context rõ → Claude trả lời chính xác hơn. Dùng Project để giữ shared context (stack, conventions), không phải để có 1 conversation khổng lồ.

🧠 Memory — Ghi nhớ về bạn

Claude tự động tổng hợp thông tin từ các conversations để cá nhân hóa phản hồi theo thời gian.

🔍
Memory hoạt động thế nào
Sau mỗi conversation, Claude trích xuất thông tin quan trọng (nghề nghiệp, preferences, tech stack) và lưu vào "memory database". Conversation sau sẽ dùng context này.
📝
Xem & chỉnh sửa Memory
Settings → Memory. Bạn thấy toàn bộ điều Claude nhớ, có thể edit hoặc xóa từng mục một cách thủ công.
🎯
Yêu cầu Claude nhớ
Nói thẳng: "Hãy nhớ tôi dùng Java 21 và Spring Boot 3.4" hay "Đừng nhớ thông tin về dự án X nữa".
🔒
Riêng tư
Memory tắt hoàn toàn trong Incognito Conversations. Tắt Memory hẳn trong Settings nếu không muốn dùng.
⚠️ Lưu ý quan trọng
Memory update có độ trễ — conversation vừa xong chưa chắc đã được lưu ngay. Khi xóa conversation, thông tin từ conversation đó sẽ được xóa khỏi Memory sau một thời gian (batch process hàng đêm).

🔌 Claude API

Tích hợp Claude vào ứng dụng của bạn qua REST API. Cần API key từ console.anthropic.com.

📞 Gọi API cơ bản

JavaScript / fetch 
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": process.env.ANTHROPIC_API_KEY,
    "anthropic-version": "2023-06-01"
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1024,
    system: "Bạn là Java architect. Trả lời bằng tiếng Việt.",
    messages: [
      { role: "user", content: "Giải thích CQRS pattern" }
    ]
  })
});

const data = await response.json();
console.log(data.content[0].text);

☕ Java / Spring Boot Integration

Java — Spring Boot 
// pom.xml: com.anthropic:sdk-java:1.x
@Service
public class ClaudeService {

    private final AnthropicClient client;

    public ClaudeService(@Value("${anthropic.api.key}") String apiKey) {
        this.client = AnthropicClient.builder()
            .apiKey(apiKey)
            .build();
    }

    public String ask(String question) {
        var message = MessageCreateParams.builder()
            .model(Model.CLAUDE_SONNET_4_6)
            .maxTokens(1024)
            .addUserMessage(question)
            .build();

        return client.messages().create(message)
            .content().get(0)
            .text().get().text();
    }
}

🔄 Streaming Response

Python — streaming 
import anthropic

client = anthropic.Anthropic()

# Stream response từng token
with client.messages.stream(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Viết hàm quicksort Python"}]
) as stream:
    for text in stream.text_stream():
        print(text, end="", flush=True)

# Lấy toàn bộ message khi stream xong
message = stream.get_final_message()
print(f"\nTotal tokens: {message.usage.input_tokens + message.usage.output_tokens}")
🎮 Demo — API parameters giải thích
reference
Parameter Type Mô tả Ví dụ
model string ID model Claude sẽ dùng "claude-sonnet-4-20250514"
max_tokens integer Số token tối đa trong response 1024
system string System prompt — hướng dẫn vai trò, ngữ cảnh "Bạn là Java expert..."
messages array Lịch sử hội thoại (user/assistant turns) [{role:"user", content:"..."}]
temperature 0.0–1.0 0 = xác định, 1 = sáng tạo hơn 0.7
stream boolean Trả về response từng chunk (SSE) true
stop_sequences array Claude dừng khi gặp chuỗi này ["END"]

🔗 MCP Servers — Kết nối công cụ ngoài

Model Context Protocol (MCP) cho phép Claude kết nối với databases, APIs, file systems và services bên ngoài.

📅
Google Calendar
Tạo, đọc, cập nhật sự kiện. "Đặt meeting với team vào thứ Hai 9h"
📧
Gmail
Đọc, tìm kiếm, soạn và gửi email. "Tóm tắt email chưa đọc hôm nay"
💾
Database (PostgreSQL, MySQL)
Query trực tiếp DB. "Đếm order tháng này theo status" — Claude tự viết và chạy SQL.
🔧
Jira / Asana
Tạo ticket, update sprint, đọc backlog. "Tạo bug ticket cho lỗi vừa tìm được"

⚙️ Cấu hình MCP trong API call

JavaScript — MCP 
const response = await fetch("https://api.anthropic.com/v1/messages", {
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1000,

      role: "user",
      content: "Tạo task trong Asana: Review PR declaration-service"
    }],
    // Khai báo MCP servers Claude có thể dùng

      type: "url",

    }]
  })

👨‍💻 Backend Developer (Java/Spring Boot)

👤
Bạn
Tôi có lỗi này khi chạy JPA query:
org.hibernate.exception.SQLGrammarException: could not extract ResultSet

Query:
@Query("SELECT o FROM Order o WHERE o.status IN :statuses") List<Order> findByStatuses(@Param("statuses") List<OrderStatus> statuses);
✦
Claude
Đây là lỗi phổ biến với Hibernate 5 khi dùng IN clause với Collection. Có 2 cách fix:

Cách 1 — Dùng @Param với Collection thẳng (recommended): 
@Query("SELECT o FROM Order o WHERE o.status IN (:statuses)")
List<Order> findByStatuses(@Param("statuses") Collection<OrderStatus> statuses);
Cách 2 — Nếu vẫn lỗi, convert sang String trước: 
List<String> statusNames = statuses.stream()
    .map(Enum::name).collect(toList());
// Rồi query với String thay vì Enum
Root cause: Hibernate 5 đôi khi không bind đúng khi parameter là List<Enum>. Upgrade lên Hibernate 6+ (Spring Boot 3.x) tự fix được vấn đề này.

📊 Data Analyst

👤
Bạn
Tôi có CSV với 3 cột: date, product_id, revenue. Viết Python script tính monthly revenue per product và plot bar chart.
✦
Claude
Đây là script hoàn chỉnh:
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('data.csv', parse_dates=['date'])
df['month'] = df['date'].dt.to_period('M')

monthly = df.groupby(['month', 'product_id'])['revenue'].sum().reset_index()
pivot = monthly.pivot(index='month', columns='product_id', values='revenue')

pivot.plot(kind='bar', figsize=(12, 6))
plt.title('Monthly Revenue by Product')
plt.xlabel('Month')
plt.ylabel('Revenue (VND)')
plt.legend(title='Product ID')
plt.xticks(rotation=45)
plt.tight_layout()
plt.savefig('revenue_chart.png', dpi=150)
plt.show()
Chạy: pip install pandas matplotlib rồi python script.py. Muốn tôi thêm filter theo date range hoặc export Excel không?

Pro tips

⭐ Mẹo nâng cao từ người dùng kinh nghiệm

🔄
Iterate aggressively
Đừng chấp nhận output đầu tiên nếu chưa ổn. "Viết lại nhưng dùng builder pattern", "Tối ưu hơn", "Ngắn hơn một nửa" — Claude không bao giờ mệt.
📌
Anchor bằng ví dụ cụ thể
Thay vì mô tả, show example: "Tôi muốn output trông như thế này: [ví dụ cụ thể]". Cực kỳ hiệu quả cho format và style nhất định.
🧩
Break down bài toán lớn
Thay vì "Xây hệ thống payment", hỏi theo từng bước: schema → service layer → API → test → deployment. Mỗi bước hoàn chỉnh trước khi tiếp.
🪞
Dùng Claude review Claude
"Review đoạn code này tôi vừa nhận được từ AI, tìm lỗi và cải thiện". Claude tốt trong việc phê bình — kể cả output của chính mình.
🎯
Explicit về constraints
"Không thêm dependency mới", "Giữ backward compatible", "Chỉ Java stdlib", "Không thay đổi interface". Claude tôn trọng constraints khi bạn nói rõ.
📚
Paste docs trực tiếp
Nếu dùng thư viện mới hoặc internal API: copy paste spec/docs vào chat trước. "Đây là API spec của service X: [...] Viết client cho nó bằng Java".
