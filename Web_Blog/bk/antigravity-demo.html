Kiến trúc & Developer Guide

Core Component

🎛 Agent Manager — Mission Control

Đây là điểm khác biệt lớn nhất của AntiGravity. Thay vì chatbox tuyến tính, Agent Manager là dashboard quản lý nhiều agent song song.

🚀
Dispatch bất kỳ lúc nào
Gửi task mới ngay khi agent khác đang chạy. Không cần chờ. Mỗi task được một agent instance riêng xử lý độc lập.
📊
Real-time status
Xem progress, artifacts đã tạo, pending approvals, và reasoning của từng agent đang chạy — tất cả trên cùng một dashboard.
📦
Artifacts-based trust
Agent không chỉ nói "đã fix" — nó tạo artifacts (plan, diff, screenshot, walkthrough) để bạn verify trước khi approve.
🌐
Workspaces & Playground
Workspace: project folder với kiểm soát chặt. Playground: scratch area để prototype nhanh, có thể convert thành workspace sau.

📋 Cách sử dụng hiệu quả

  • Dùng @filename và @foldername để thêm context vào task
  • Planning mode cho complex tasks (tổ hợp) — agent sẽ tạo task list và implementation plan artifact (Tạo tác kế hoạch thực hiện) trước khi code
  • Fast mode cho simple tasks như rename variables, format code, chạy terminal command
  • Review artifact ở từng bước — đừng auto-approve toàn bộ, đây là lúc phát hiện sai hướng sớm nhất
  • Sau khi review, comment trực tiếp vào artifact nếu cần agent điều chỉnh
Core Component

📝 Editor View

Editor View là môi trường coding quen thuộc kế thừa từ VS Code — với AI được tích hợp sâu hơn plugin thông thường.

⌨️
Tab Completions
AI autocomplete inline, context-aware từ toàn bộ workspace. Hiểu codebase của bạn, không chỉ file đang mở.
💬
Inline Commands
Select code → Cmd+K để ra lệnh trực tiếp. "Refactor this to use async/await", "Add error handling" — agent sửa ngay tại chỗ.
🔄
Bidirectional
Bạn edit file trong Editor, agent thấy changes ngay. Agent tạo file mới, bạn thấy trong Editor ngay. Hai chiều real-time.
💡 Editor vs Agent Manager — khi nào dùng gì?
Agent Manager: task phức tạp, multi-file, cần plan, cần verify qua browser. Editor View: đang hands-on code, cần gợi ý inline nhanh, hoặc chỉnh sửa trực tiếp file mà agent vừa tạo.
Core Component

🌐 Browser Subagent

Tính năng độc đáo nhất của AntiGravity — agent có thể thực sự điều khiển trình duyệt để test và verify, không phải giả lập.

🖱
Full browser control
Click buttons, fill forms, scroll, navigate — agent điều khiển Chrome thật sự qua Chrome Extension được cài riêng.
👁
Đọc & hiểu trang
DOM capture, screenshot, markdown parsing — agent "đọc" được trang web, không chỉ click mù quáng.
🧪
Automated testing
Agent tự deploy app, mở browser, test từng flow, chụp screenshot làm Walkthrough artifact. Bạn review video-like walkthrough thay vì đọc code.
📹
Browser recordings
Agent tạo recording (video ngắn) của toàn bộ test session — đây là artifact dạng Walkthrough giúp bạn verify kết quả trực quan.
⚠️ JavaScript Execution Policy
Browser subagent có thể chạy JavaScript trong trang. Request Review (recommended) — agent hỏi permission trước khi chạy JS. Always Proceed — tự động nhưng có risk Prompt Injection. Disabled — agent không chạy JS, hạn chế khả năng automation.
Core Concept

📦 Artifacts — Hệ thống "Bằng chứng"

Artifacts là cơ chế giải quyết "Trust Gap" — thay vì tin lời agent nói, bạn có bằng chứng cụ thể để verify. Đây là innovation quan trọng nhất của AntiGravity về mặt UX.

Artifact TypeKhi nào xuất hiệnNội dungBạn cần làm gì
📋 Task List Ngay khi agent nhận task (Planning mode) Danh sách subtasks có cấu trúc, dependency, ước tính thời gian Review, comment nếu cần điều chỉnh direction
🏗 Implementation Plan Sau Task List, trước khi code Technical details: files sẽ thay đổi, approach cụ thể, tradeoffs Review kỹ nhất ở đây — dễ phát hiện sai hướng nhất
📄 Code Diff Sau khi agent viết/sửa code Diff của tất cả files thay đổi Review changes, approve hoặc request chỉnh sửa
🖼 Screenshot Sau browser test Ảnh chụp UI trước/sau, kết quả test Verify UI trông đúng như mong đợi
🎬 Walkthrough Sau E2E testing Recording của agent test flow trên browser thật Xem như xem video demo — verify full flow hoạt động
📊 Architecture Diagram Khi planning complex system Sơ đồ kiến trúc, data flow, component diagram Review và comment ngay trên diagram
⭐ Pro Tip — Artifact-first thinking
Đừng skip artifact review để "code nhanh hơn". Thời gian review Implementation Plan là thời gian tiết kiệm được từ việc phải undo sai hướng sau đó. Rigorous interrogation tại Artifact stage = chất lượng output cao hơn.
Customization #1

📜 Rules — Hướng dẫn bất biến cho Agent

Rules là system instructions luôn chạy ngầm — agent đọc và tuân theo mọi lúc, không cần bạn nhắc lại. Tương tự CLAUDE.md nhưng cho Gemini.

🌍
Global Rules
Áp dụng cho mọi project bạn mở. Định nghĩa triết lý coding cá nhân hoặc tổ chức. Vị trí: ~/.gemini/GEMINI.md
📁
Workspace Rules
Áp dụng chỉ cho project hiện tại, override/supplement Global Rules. Vị trí: .agent/rules/ trong project root

📄 Ví dụ: ~/.gemini/GEMINI.md — Global Rules

Markdown · ~/.gemini/GEMINI.md
# Global Rules — Personal Coding Philosophy

## Language
- Luôn dùng TypeScript thay vì JavaScript
- Python: dùng type hints bắt buộc (Python 3.10+)
- Không bao giờ hardcode API keys hoặc secrets

## Code Quality
- Luôn thêm docstring/JSDoc cho public methods
- Error handling: không bao giờ silent catch — luôn log hoặc rethrow
- Tên biến: descriptive, không viết tắt trừ khi rất phổ biến (i, url, db)

## Testing
- Viết test trước hoặc song song với code (không phải sau)
- Test file đặt cạnh source file: foo.ts → foo.test.ts

## Security
- Không generate hardcoded credentials dù là trong example code
- SQL: luôn dùng parameterized queries, không string concatenation

📄 Ví dụ: .agent/rules/backend.md — Workspace Rules

Markdown · .agent/rules/backend.md
# Backend Rules — Payment Service

## Stack
- Runtime: Node.js 22 + TypeScript strict mode
- Framework: Fastify (không dùng Express)
- DB: PostgreSQL với Prisma ORM
- Auth: JWT + refresh tokens

## Architecture
Dùng Layered Architecture:
- routes/     ← HTTP layer (thin, chỉ parse/validate)
- services/   ← Business logic
- repositories/ ← Database access
- domain/     ← Types, interfaces, value objects

## API Design
- Response format: { data, error, meta: { requestId, timestamp } }
- Versioning: /api/v1/ prefix
- Validation: Zod cho tất cả input

## Error Handling
- Custom AppError class với code + statusCode
- Không trả về stack trace trong production
- Log level: error → Winston, format JSON
💡 Rules vs Workflows
Rules = passive, luôn chạy ngầm, không cần trigger. Giống system prompt. Workflows = active, chỉ chạy khi bạn gọi bằng /slash command. Giống saved prompt templates.
Customization #2

🔄 Workflows — Quy trình có thể gọi lại

Workflows là saved prompts trigger-on-demand bằng /slash command. Định nghĩa một lần, dùng mãi mãi. Có thể chain workflows lồng nhau.

🌍
Global Workflows
Dùng được trong mọi project. Vị trí: ~/.gemini/antigravity/global_workflows/
📁
Workspace Workflows
Chỉ dùng trong project hiện tại. Vị trí: .agent/workflows/ trong project root

📄 Ví dụ: .agent/workflows/generate-tests.md

Markdown · .agent/workflows/generate-tests.md
# Workflow: generate-tests
# Trigger: /generate-tests
# Mô tả: Tạo unit tests cho file đang mở

1. Đọc file source đang được focus trong editor
2. Xác định tất cả public functions/methods cần test
3. Với mỗi function, tạo test cases cho:
   - Happy path (input hợp lệ)
   - Edge cases (null, empty, boundary values)
   - Error cases (invalid input, exceptions)
4. Tạo file test cạnh source file: {name}.test.ts
5. Chạy tests bằng terminal: npm test -- {filename}
6. Nếu có test fail, phân tích và fix
7. Tạo Test Summary artifact hiển thị coverage và kết quả

📄 Ví dụ: .agent/workflows/code-review.md

Markdown · .agent/workflows/code-review.md
# Workflow: review
# Trigger: /review
# Mô tả: Code review toàn diện cho staged changes

1. Chạy `git diff --staged` để xem changes
2. Phân tích theo thứ tự ưu tiên:
   - 🔴 Security issues (injection, auth bypass, data exposure)
   - 🔴 Correctness bugs (logic errors, off-by-one, null deref)
   - 🟡 Performance issues (N+1 queries, unnecessary computation)
   - 🟡 Code style violations (theo .agent/rules/)
   - 🟢 Suggestions (refactoring, naming, documentation)
3. Tạo Review Report artifact với format:
   ```
   ## Summary
   [Tổng quan: X critical, Y warnings, Z suggestions]
   
   ## Issues
   [Danh sách từng issue với file:line, severity, explanation, fix]
   ```
4. Nếu có critical issues: đề xuất fix code cụ thể

📄 Ví dụ: .agent/workflows/idea-to-app.md (từ Google Cloud Solutions)

Markdown · .agent/workflows/idea-to-app.md
# Workflow: idea-to-app
# Trigger: /idea-to-app
# Dùng cho: Prototype một web app hoàn chỉnh từ mô tả

1. Hỏi: "Mô tả web app bạn muốn build (1-2 câu)"
2. Tạo Architecture Diagram artifact thể hiện:
   - Components chính
   - Data flow

   - API endpoints

✅ Chain Workflows
Workflows có thể gọi workflow khác. Ví dụ: workflow /ship-feature có thể include: "Chạy /generate-tests trước, sau đó chạy /review, nếu pass thì tạo PR". Tạo automation pipeline phức tạp chỉ bằng Markdown.
Customization #3

⚡ Skills — Progressive Disclosure Knowledge

Skills giải quyết vấn đề "tool bloat" — thay vì load tất cả context vào agent ngay từ đầu (gây chậm, tốn token, confuse), Skills chỉ được load khi request match với skill's description.

🔑 Nguyên lý Progressive Disclosure
Agent biết general programming (từ Gemini 3). Skill cung cấp specialized knowledge của bạn: project conventions, business rules, domain expertise. Khi user hỏi "run database migration", agent scan available skills, tìm database-migration skill, load nó vào context và thực thi đúng theo team standard.
🌍
Global Skills
Dùng được mọi nơi, ví dụ: "Format JSON", "Generate changelog". Vị trí: ~/.gemini/antigravity/skills/
📁
Workspace Skills
Riêng cho project, ví dụ: skill biết về domain model của bạn. Vị trí: .agent/skills/ trong project root

name: database-migration

  Khi nào LOAD skill này: khi user nhắc đến "migration", "migrate database",

# Skill: Database Migration

## Project DB Stack
- ORM: Prisma với PostgreSQL 16
- Migration files trong: prisma/migrations/
- Staging DB: postgresql://staging.internal/appdb

## Quy trình tạo migration
1. Cập nhật prisma/schema.prisma với thay đổi cần thiết
2. Chạy: `npx prisma migrate dev --name {descriptive-name}`
3. Review generated SQL trong prisma/migrations/{timestamp}_{name}/
4. Test trên staging trước: `DATABASE_URL=$STAGING npx prisma migrate deploy`
5. Chỉ deploy production khi staging OK

## Naming convention
Migration name: snake_case, mô tả rõ ràng
Ví dụ: add_user_email_index, create_payments_table, drop_legacy_sessions

## Không được làm
- Không edit migration file sau khi đã commit
- Không xóa migration files cũ (dù đã applied)
- Không dùng raw SQL trong code — luôn qua Prisma client

🔌 MCP Tools — Kết nối services bên ngoài

AntiGravity hỗ trợ Model Context Protocol (MCP) — agent có thể gọi tools kết nối với databases, GitHub, Jira, và bất kỳ service nào có MCP server.

🐙
GitHub MCP
Đọc issues, tạo PR, xem repo structure. Agent tự tạo PR sau khi implement feature.
🗄️
Database MCP
Query DB trực tiếp để lấy context. Agent biết schema thực tế, không cần bạn paste.
📋
Jira / Linear MCP
Đọc tickets, cập nhật status. "Fix bug từ ticket PROJ-123" — agent đọc description tự động.

Security

🛡 Agent Security

Khi agent có quyền truy cập terminal và browser, security trở thành critical. AntiGravity có hệ thống phân quyền chi tiết.

✅
Allow List
Danh sách commands/domains agent được phép dùng không cần hỏi. Thêm vào ~/.gemini/antigravity/browserAllowlist.txt.
🚫
Deny List
Danh sách commands agent không bao giờ được chạy — dù ở mode "Always Proceed". Ví dụ: rm -rf, DROP TABLE.
💉
Prompt Injection Risk
Khi agent đọc website hoặc file từ external source, content đó có thể chứa instructions giả mạo. Dùng "Request Review" mode để kiểm soát.
📤
Data Exfiltration Risk
Agent có thể gọi external API nếu không có restrict. Trong Secure Mode, agent không thể gửi data ra ngoài domain allowlist.
🚨 Nguyên tắc an toàn
Không dùng Agent-driven mode cho production codebases. Luôn dùng Review-driven hoặc Secure mode. Cấu hình deny list cho các lệnh nguy hiểm. Không cấp GITHUB_TOKEN production vào MCP — dùng token có scope giới hạn nhất.

Agent Modes

⚙️ Planning Mode vs Fast Mode

Hai mode quyết định agent "suy nghĩ" bao nhiêu trước khi hành động — ảnh hưởng trực tiếp đến chất lượng và tốc độ.

🧠
Planning Mode (default)
Agent suy nghĩ sâu, tạo Task List → Implementation Plan artifacts trước khi viết bất kỳ dòng code nào. Tổ chức work thành task groups. Output chất lượng cao hơn nhiều.

Dùng khi: feature mới, refactoring phức tạp, architecture decisions, bất kỳ task >30 phút.
⚡
Fast Mode
Agent thực thi trực tiếp, không tạo plan artifacts. Nhanh hơn đáng kể.

Dùng khi: rename variable, fix typo, format code, thêm một câu log, chạy bash commands đơn giản, tasks rõ ràng có thể hoàn thành trong <5 phút.
💡 Mental model
Planning Mode = "Thinking budget cao" — agent dùng nhiều compute để reason trước khi act. Fast Mode = "Thinking budget thấp" — execute nhanh. Tương tự DeepThink vs standard mode trong Gemini API.

Best Practices

⭐ Best Practices từ Google & Community

🏗
Đừng skip Artifact review
Artifact-first workflow
Interrogate kỹ Implementation Plan trước khi cho agent code. Sai hướng phát hiện ở plan = tiết kiệm nhiều giờ undo sau này.
📝
Rules ngắn gọn, cụ thể
Rules best practice
Rules quá dài làm loãng context. Ưu tiên rules quan trọng nhất, checkable được. "Không dùng var" tốt hơn "Viết code tốt".
⚡
Workflow cho tasks lặp lại
Productivity
Bất kỳ task bạn làm hơn 2 lần/tuần đều nên có workflow. /generate-tests, /review, /create-pr là bộ 3 cơ bản nhất.
🎯
Mô tả task như kiến trúc sư
Prompting
"Build a user auth system with JWT, refresh tokens, role-based access" tốt hơn "viết code login". Bạn là architect, agent là engineer.
🔒
Luôn dùng Review-driven
Security
Agent-driven tiện nhưng nguy hiểm trên codebase thật. Review-driven cho bạn checkpoint approve trước mỗi hành động quan trọng.
🧠
Dùng @context reference
Context management
@file.ts, @folder/, @README.md — reference trực tiếp thay vì paste content. Agent đọc trực tiếp từ workspace, chính xác hơn.
⭐ Core Philosophy
AntiGravity không thay thế developer — nó amplify. Bạn chuyển từ "người viết code" sang "người chỉ đạo". Đầu tư vào Rules, Workflows, Skills = đầu tư vào năng suất dài hạn. Agent quality = context quality.
