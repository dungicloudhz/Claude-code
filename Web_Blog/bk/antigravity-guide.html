# Global Rules — Google AntiGravity Project

## ❌ TUYỆT ĐỐI KHÔNG
- Không hardcode API keys, secrets vào code — dùng Orbital Vault (secret manager)
- Không gọi AntiGravity SDK trực tiếp từ UI component — phải qua service layer
- Không dùng floating-point cho tọa độ orbital — dùng OrbitalCoordinate value type
- Không mutation state trọng lực trực tiếp — phải qua GravityReducer
- Không deploy lên production mà không chạy gravity-test suite

## ✅ BẮT BUỘC
- Mọi API call phải có timeout: 5s cho read, 30s cho write
- Logging: structured JSON, include traceId và orbitalSessionId
- Error messages: tiếng Anh trong code, tiếng Việt trong UI
- Test: 80% coverage cho domain, 60% cho application layer
- PR: phải có screenshot/recording nếu thay đổi 3D behavior

## 📏 NAMING
- File: camelCase cho JS/TS, PascalCase cho components
- Orbital entities: prefix O (OStation, OModule, OTrajectory)
- API endpoints: /api/v2/orbital/{resource}
- Event names: ORBITAL_ENTITY_ACTION (STATION_MODULE_ATTACHED)
- Constants: UPPER_SNAKE trong dedicated constants file

# Backend Rules — Node.js / TypeScript

## Architecture
Dùng Hexagonal Architecture: domain/ → application/ → adapter/
- domain/: pure TypeScript, không import từ Express/Fastify
- application/: use cases, orchestration
- adapter/: controllers, repositories (implementation)

## TypeScript
- Strict mode: true — không được bật any
- Không dùng 'as' type assertion trừ khi không thể tránh
- Return types phải explicit cho public functions
- Zod cho tất cả external data validation (API input, DB output)

## Database (Nebula DB)
- Tất cả queries phải có TTL hoặc pagination
- Không bao giờ query toàn collection không có index filter
- Transaction: dùng NebulaTx.atomic() cho multi-document updates
- Connection pooling: max 20 connections qua OrbitalDbPool

## API Design
- REST theo Orbital API Spec v2 (xem docs/orbital-api-spec.md)
- Response wrapper: { success, data, meta: { requestId, timestamp } }
- Error: { success: false, error: { code, message, details } }
- Rate limiting: 100 req/min per orbital session

# Skill: Orbital API Integration

## Khi nào dùng
Khi cần tương tác với AntiGravity platform: tạo/move orbital objects,
subscribe events, query spatial data, control gravity zones.

## AntiGravity SDK — TypeScript

```typescript
import { AntiGravityClient, OrbitalObject, GravityZone } from '@google/antigravity-sdk';

const ag = new AntiGravityClient({
  apiKey: process.env.AG_API_KEY,  // từ Orbital Vault
  region: 'asia-orbital-1',
  gravityProfile: 'earth-standard',
});

// Tạo orbital object
const module = await ag.orbital.create({
  type: 'OModule',
  coordinates: new OrbitalCoordinate(0, 400, 0),  // km từ ground
  mass: 500,  // metric tons
  attachPoint: stationId,
});

// Subscribe to orbital events (WebSocket)
ag.events.subscribe('GRAVITY_SHIFT', (event) => {
  console.log('Gravity changed:', event.newGravity);
});

// Query spatial (NOT: don't use string coordinates)
const nearby = await ag.spatial.query({
  center: OrbitalCoordinate.fromLatLng(-90, 0, 400),
  radius: 50,  // km
  type: ['OStation', 'OModule'],
});
```

## Anti-patterns
- Không dùng số float trực tiếp cho coordinates — dùng OrbitalCoordinate
- Không poll API để theo dõi events — dùng WebSocket subscription
- Không cache orbital positions lâu hơn 1 giây — chúng liên tục thay đổi

# Skill: AntiGravity 3D Components

## Pattern chuẩn
AntiGravity dùng React với @google/ag-ui-kit cho 3D components.
Mọi 3D component phải wrap trong <OrbitalCanvas>.

```tsx
import { OrbitalCanvas, Station3D, GravityZone, useOrbital } from '@google/ag-ui-kit';

export function OrbitalDashboard() {
  const { objects, isLoading } = useOrbital({ type: 'OStation' });

  return (
    <OrbitalCanvas
      fov={75}
      gravityProfile="earth-standard"
      onObjectClick={(obj) => navigate(`/station/${obj.id}`)}
    >
      {/* Background stars */}
      <StarField density={1000} />

      {/* Earth */}
      <Planet type="earth" position={[0, 0, -6371]} />

      {/* Orbital stations */}
      {objects.map((station) => (
        <Station3D
          key={station.id}
          coordinates={station.coordinates}
          status={station.status}
          label={station.name}
        />
      ))}

      {/* Gravity zones */}
      <GravityZone radius={500} strength={9.8} visible={true} />
    </OrbitalCanvas>
  );
}
```

## Performance rules
- Max 500 objects trong một scene — dùng LOD (Level of Detail) nếu cần hơn
- Không animate tất cả objects cùng lúc — dùng instanced rendering
- Texture: max 2K resolution, KTXTWO format (compressed)
- useOrbital hook tự động unsubscribe khi component unmount

# Known Issues & Gravity Anomalies

## [AG-001] OrbitalCoordinate precision loss > 1000km altitude
**Symptom:** Objects drift từ 0.001km đến ~50m sau 24h orbit
**Root cause:** Float64 precision không đủ cho tọa độ orbital dài hạn
**Workaround:** Dùng OrbitalCoordinate.highPrecision() cho altitude > 1000km
**Status:** Fix trong SDK 3.2, hiện đang dùng 3.1

---

## [AG-002] WebSocket disconnect khi gravity shift event
**Symptom:** ag.events.subscribe() ngắt kết nối khi gravity > 12.0 m/s²
**Root cause:** Event payload quá lớn (gravity shift kéo theo cascade events)
**Workaround:** Subscribe riêng: GRAVITY_SHIFT_STARTED và GRAVITY_SHIFT_ENDED
**Note:** KHÔNG subscribe GRAVITY_SHIFT event trực tiếp

---

## [AG-003] OrbitalCanvas re-render toàn bộ khi 1 object update
**Symptom:** FPS drop từ 60 xuống 8 khi nhiều objects update đồng thời
**Root cause:** Missing key prop hoặc không dùng useMemo cho static objects
**Fix:**
```tsx
// ❌ Sai
{objects.map(o => <Station3D {...o} />)}

// ✅ Đúng
{objects.map(o => <Station3D key={o.id} {...o} />)}
// Và wrap OrbitalCanvas children trong useMemo nếu static
```

---

## [AG-004] Nebula DB timeout với filter phức tạp
**Symptom:** query_nebula timeout sau 5s với nested $and/$or filter
**Workaround:** Chia thành nhiều query đơn giản hơn + manual filter trong JS
**Index cần thêm:** Đã tạo ticket AG-INFRA-445

# Architectural Decisions

## [ADR-001] Fastify thay vì Express
**Date:** 2025-10-01 | **Status:** Accepted
**Decision:** Dùng Fastify 5 cho API server
**Reason:** Performance tốt hơn 2x, built-in TypeScript support,
schema validation với Zod tích hợp tốt hơn.
**Note:** Claude sẽ không suggest Express trong bất kỳ context nào.

## [ADR-002] OrbitalCoordinate value type thay vì number[]
**Date:** 2025-10-15 | **Status:** Accepted
**Decision:** Tất cả tọa độ spatial phải dùng OrbitalCoordinate class
**Reason:** Tránh AG-001 precision bug, enforce unit consistency (km, không m).
**Rule:** Bắt buộc trong .ai/rules/global.md

---

## [ADR-003] Feature-Sliced Design cho Frontend

☕ Backend hoàn chỉnh — Docking API

Feature: API cho phép OModule dock vào OStation — từ domain model đến endpoint, theo Hexagonal Architecture.

1. Domain Model

TypeScript · domain/model/OStation.ts 
// domain/model/OStation.ts — Aggregate Root
// ✅ Pure TypeScript — no Fastify, no Nebula imports
export class OStation {
  readonly id: StationId;
  private coordinates: OrbitalCoordinate;
  private dockedModules: Map<ModuleId, OModule>;
  private status: StationStatus;
  private readonly maxModules = 12;

  // Factory method
  static create(coords: OrbitalCoordinate, name: string): OStation {
    return new OStation(StationId.generate(), coords, name, 'ACTIVE');
  }

  // Domain logic — docking business rules
  dockModule(module: OModule): void {
    if (this.status !== 'ACTIVE') {
      throw new DomainError(`Station ${this.id} không active`);
    }
    if (this.dockedModules.size >= this.maxModules) {
      throw new DomainError(`Station đã đầy (${this.maxModules} modules)`);
    }
    if (!this.isProximityOk(module.coordinates)) {
      throw new DomainError('Module quá xa để dock (>10km)');
    }
    this.dockedModules.set(module.id, module);
  }

  private isProximityOk(coord: OrbitalCoordinate): boolean {
    return this.coordinates.distanceTo(coord) <= 10; // km
  }
}

2. Use Case + Route

TypeScript · adapter/in/rest/docking.route.ts 
// Zod schema — validate input trước khi vào domain
const DockModuleBodySchema = z.object({
  moduleId: z.string().uuid(),
  initiatorId: z.string().uuid(),
});

// Fastify route — adapter layer (NOT domain)
export const dockingRoute: FastifyPluginAsync = async (fastify) => {
  fastify.post<{ Params: { stationId: string }; Body: DockModuleBody }>(
    '/api/v2/orbital/stations/:stationId/dock',
    {
      schema: {
        params: z.object({ stationId: z.string().uuid() }),
        body: DockModuleBodySchema,
      },
      preHandler: [fastify.authenticate],  // Orbital IAM check
    },
    async (req, reply) => {
      const command = new DockModuleCommand({
        stationId: StationId.of(req.params.stationId),
        moduleId: ModuleId.of(req.body.moduleId),
        initiatorId: req.body.initiatorId,
      });

      const result = await dockModuleUseCase.execute(command);

      return reply.code(200).send({
        success: true,
        data: result,
        meta: { requestId: req.id, timestamp: new Date().toISOString() },
      });
    }
  );
};

Hook + 3D Component

TypeScript · features/stations/hooks/useStation.ts 
// React Query hook cho station data
export const stationKeys = {
  all: ['stations'] as const,
  detail: (id: string) => [...stationKeys.all, id] as const,
};

export function useStation(id: string) {
  return useQuery({
    queryKey: stationKeys.detail(id),
    queryFn: () => stationsApi.getById(id),
    staleTime: 1000,  // 1s — orbital positions change fast
    refetchInterval: 2000,  // poll mỗi 2s cho real-time updates
  });
}

export function useDockModule() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: stationsApi.dock,
    onSuccess: (_, vars) => {
      qc.invalidateQueries({ queryKey: stationKeys.detail(vars.stationId) });
      toast.success('🛸 Module đã dock thành công!');
    },
    onError: (e: OrbitalError) => {
      toast.error(`Docking thất bại: ${e.message}`);
    },
  });
}
TypeScript · features/stations/components/StationView3D.tsx 
// Named export — không default export (theo rules)
export function StationView3D({ stationId }: { stationId: string }) {
  const { data: station, isLoading } = useStation(stationId);
  const dock = useDockModule();

  if (isLoading) return <OrbitalLoadingScreen message="Locking orbital coordinates..." />;

  return (
    <OrbitalCanvas
      fov={60}
      gravityProfile="earth-standard"
      initialPosition={station?.coordinates}
      onObjectClick={(obj) =>
        dock.mutate({ stationId, moduleId: obj.id })
      }
    >
      <StarField density={800} />
      <Earth position={[0, 0, -6371]} />

      {/* Station */}
      <Station3D
        coordinates={station?.coordinates}
        status={station?.status}
        label={station?.name}
        highlight={dock.isPending}
      />

      {/* Docked modules */}
      {station?.modules.map((m) => (
        <Module3D key={m.id} module={m} />
      ))}

      {/* Gravity zone indicator */}
      <GravityZone radius={50} strength={9.8} visible opacity={0.15} />
    </OrbitalCanvas>
  );
}

# CLAUDE.md — Google AntiGravity Platform
> Stack: Node.js 22 + TypeScript + React 19 + AntiGravity SDK 3.1
> Region: asia-orbital-1 | Gravity: earth-standard (9.8 m/s²)

## 🌌 Dự án là gì?
Nền tảng quản lý orbital stations và modules trong không gian ảo.
Users: aerospace engineers, space logistics teams. ~2000 MAU.
