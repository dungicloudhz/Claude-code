OpenClaw — AI Agent cá nhân chạy trên thiết bị của bạn

OpenClaw (trước đây là ClawdBot, MoltBot) là một personal AI agent mã nguồn mở bạn tự host. Không giống ChatGPT hay Claude.ai — bạn chat với nó qua WhatsApp, Telegram, Discord, SMS... nơi bạn đang dùng hàng ngày. Nó nhớ bạn, tự động hoá tác vụ, và có thể thực thi lệnh thật sự trên máy tính.

  "auth": {
    "type": "token",

  "tools": {
    "allow": ["group:fs", "browser", "web_search", "exec"],

  "providers": {
    "ollama": {
      "baseUrl": "http://localhost:11434",
      "model":   "llama3.3:70b"
    }
  }
}

Kiểm soát Tool với Allow/Deny Lists
openclaw.json — kiểm soát tool permissionsTOOLS CONFIG
"tools": {
  // Profile đặt base allowlist
  "profile": "full",      // full | coding | messaging | minimal

  // Cho phép thêm các tool/group
  "allow": [
    "group:fs",           // read, write, edit, apply_patch
    "group:web",          // web_search, web_fetch
    "group:runtime",      // exec, bash, process
    "browser",            // Chrome automation
    "image"               // image analysis/generation
  ],

  // Cấm — deny luôn thắng allow
  "deny": [
    "exec"                // cấm chạy shell (an toàn hơn)
  ],

  // Hạn chế theo provider cụ thể
  "byProvider": {
    "google": { "profile": "minimal" }
  }
}

// Tool Groups có sẵn:
// group:runtime   → exec, bash, process
// group:fs        → read, write, edit, apply_patch
// group:sessions  → sessions_list, sessions_history, sessions_send
// group:memory    → memory_search, memory_get
// group:web       → web_search, web_fetch
// group:ui        → browser, canvas
// group:messaging → message

# Daily Standup Skill

Khi người dùng yêu cầu "daily standup" hoặc "báo cáo sáng":

## Quy trình:
1. Dùng `web_fetch` để lấy Jira tickets assigned cho user hôm nay
2. Dùng `read` để đọc file notes từ hôm qua tại `{baseDir}/yesterday.md`
3. Dùng `web_search` để check breaking news liên quan đến dự án
4. Tổng hợp thành format standup:

```
✅ Hôm qua làm gì?
🎯 Hôm nay plan gì?
🚫 Có blocker gì không?
```

5. Gửi qua `message` đến channel Slack team nếu user yêu cầu

## Lưu ý:
- Luôn hỏi user có muốn gửi cho team không (không tự gửi)
- Format ngắn gọn < 200 chữ
- Highlight item urgent bằng 🚨
Ví dụ skill thực tế — Theo dõi giá cổ phiếu
workspace/skills/stock-tracker/SKILL.mdEXAMPLE
---
name: stock-tracker
description: Theo dõi giá cổ phiếu và cảnh báo khi biến động mạnh
metadata: {"openclaw": {"emoji": "📈", "requires": {"bins": ["curl"]}}}
---

# Stock Tracker Skill

Khi user hỏi về giá cổ phiếu hoặc yêu cầu theo dõi:

1. Dùng `web_fetch` gọi API để lấy giá: `https://query1.finance.yahoo.com/v8/finance/chart/{TICKER}`
2. Parse JSON response để lấy: current price, change%, volume
3. So sánh với giá user đặt alert (lưu trong `{baseDir}/watchlist.json`)
4. Nếu biến động > 5%: gửi alert với emoji 📈/📉

Watchlist format trong watchlist.json:
```json
[
  {"ticker": "VIC", "alert_threshold": 5, "buy_price": 45000},
  {"ticker": "VNM", "alert_threshold": 3, "buy_price": 72000}
]
```

Khi user nói "thêm VIC vào watchlist với alert 5%" → cập nhật watchlist.json.
Khi user nói "xem watchlist" → fetch giá hiện tại cho tất cả và so sánh với buy_price.

Cron Jobs & Scheduler

Tạo lịch tự động, one-shot reminder, recurring jobs, gửi kết quả về channel

Cách hoạt động — 3 thành phần
📅
Schedule
Khi nào chạy: at (một lần), every (định kỳ ms), cron (cron expression 5-6 field).
💬
Payload
Làm gì: systemEvent (inject vào main session) hoặc agentTurn (isolated session riêng).

OpenClawService.java — Gọi OpenClaw từ Spring BootSPRING BOOT
@Service
public class OpenClawService {

    private final WebClient webClient;

    public OpenClawService(WebClient.Builder builder) {
        this.webClient = builder
            .baseUrl("http://localhost:18789") // Gateway URL
            .defaultHeader("Authorization", "Bearer your-token")
            .build();
    }

    // Gửi message đến agent, nhận phản hồi
    public Mono<String> askAgent(String message) {
        return webClient.post()
            .uri("/api/v1/chat")
            .bodyValue(Map.of("message", message, "session", "main"))
            .retrieve()
            .bodyToMono(ChatResponse.class)
            .map(ChatResponse::getText);
    }

    // Tạo cron job từ Java code
    public Mono<String> scheduleDailyReport(String userId) {
        var cronRequest = Map.of(
            "name",    "Daily Report - " + userId,
            "cron",    "0 8 * * *",
            "tz",      "Asia/Ho_Chi_Minh",
            "session", "isolated",
            "message", "Tạo daily report cho user " + userId
        );
        return webClient.post()
            .uri("/api/v1/cron")
            .bodyValue(cronRequest)
            .retrieve()
            .bodyToMono(String.class);
    }

    // Streaming response (SSE)
    @GetMapping("/stream-ai")
    public SseEmitter streamFromOpenClaw(String prompt) {
        SseEmitter emitter = new SseEmitter(30_000L);
        webClient.post().uri("/api/v1/chat/stream")
            .bodyValue(Map.of("message", prompt))
            .retrieve()
            .bodyToFlux(String.class)
            .subscribe(emitter::send, emitter::completeWithError, emitter::complete);
        return emitter;
    }
}

Bảo mật & Docker Sandbox

Auth token, pairing, sandbox, không expose Gateway ra internet

Các lớp bảo mật của OpenClaw
🔑
Auth Token
Gateway yêu cầu token khi kết nối. Chỉ ai có token mới chat được.
🤝
Pairing
Thiết bị phải được "pair" với Gateway. Không pair = không nhận lệnh.
📦
Sandbox
Docker sandbox isolate agent khỏi host OS. Tool exec chạy trong container riêng.
🚫
Tool Deny List
Cấm tools nguy hiểm qua deny list. Deny luôn thắng allow.
Docker Sandbox — Chạy agent an toàn
openclaw.json — Docker sandbox configSECURITY
{
  "agents": {
    "defaults": {
      "sandbox": {
        "enabled": true,
        "docker": {
          "image": "openclaw/sandbox:latest",
          // Chạy 1 lần khi tạo container — cài tools cần thiết
          "setupCommand": "apt-get update && apt-get install -y curl jq",
          // Giới hạn tài nguyên
          "memory": "512m",
          "cpus":   "1.0",
          // Mount workspace vào container (read-write)
          "mounts": [
            "~/.openclaw/workspace:/workspace:rw"
          ]
        }
      }
    }
  }
}

🚨
KHÔNG expose Gateway trực tiếp ra Internet! Port 18789 chỉ nên accessible từ localhost hoặc qua reverse proxy có auth. Khi deploy trên VPS, dùng nginx + SSL + auth trước Gateway. Nếu dùng Telegram/Discord, bot kết nối ra ngoài — không cần port forwarding.
