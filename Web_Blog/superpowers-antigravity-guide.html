Vấn đề

😤 Tại sao cần Superpowers?

Mặc định, Gemini trong Antigravity sẽ bắt đầu viết code ngay khi bạn gõ yêu cầu — không hỏi, không lên kế hoạch, không kiểm tra. Điều này dẫn đến:

❌ Không có Superpowers
Bạn: "Build me a CLI tool"
AI: bắt đầu code ngay lập tức...
Bạn: nhận ra giữa chừng không phải thứ mình muốn

→ Lãng phí thời gian, code phải viết lại từ đầu
✅ Có Superpowers
Bạn: "Build me a CLI tool"
→ Brainstorm: AI hỏi câu hỏi làm rõ
→ Plan: AI viết plan từng bước, bạn approve
→ Execute: AI build từng bước, có tests
→ Review: AI kiểm tra lại toàn bộ
💡 Core Idea
Superpowers không làm Gemini thông minh hơn — nó làm Gemini có kỷ luật hơn. Framework cung cấp rules, workflows và skills để AI luôn lên kế hoạch trước, viết tests, và kiểm tra kết quả thay vì chạy ngay.
Bước 1 · Cài đặt

📦 Cài đặt nhanh

Superpowers chỉ là một thư mục .agent/ mà bạn copy vào project. Không cần npm install, không cần build.

① Clone repo

bash 
# Clone repo Superpowers về máy
git clone https://github.com/anthonylee991/gemini-superpowers-antigravity.git

# Di chuyển vào thư mục clone
cd gemini-superpowers-antigravity

② Copy .agent/ vào project của bạn

Copy thư mục .agent/ vào root của project bạn đang làm việc:

bash — macOS / Linux 
# Thay /path/to/gemini-superpowers-antigravity bằng đường dẫn thực tế
cp -r /path/to/gemini-superpowers-antigravity/.agent /path/to/your-project/
PowerShell — Windows 
# Thay đường dẫn phù hợp
Copy-Item -Recurse C:\path\to\gemini-superpowers-antigravity\.agent C:\your-project\

③ Mở project trong Antigravity

1
Mở Antigravity IDE
Chọn File → Open Folder và chọn thư mục project của bạn (thư mục chứa .agent/).
2
Kiểm tra Gemini CLI đã cài
Mở terminal trong Antigravity và chạy: gemini --version. Nếu thấy số version → OK. Nếu không, cài Gemini CLI trước.
3
Reload Superpowers
Chat với Gemini và gõ: /superpowers-reload. AI sẽ xác nhận đã load thành công với danh sách workflows và skills.
✅ Xác nhận cài đặt thành công
Sau khi gõ /superpowers-reload, Gemini sẽ phản hồi:

"I've reloaded the Superpowers framework: Rules: superpowers.md / Workflows: 8 loaded / Skills: 9 loaded — I will follow these instructions for the rest of this session."

Workflow 02

📋 /superpowers-write-plan

Workflow quan trọng nhất. Buộc AI tạo plan chi tiết trước khi viết một dòng code nào. Plan được ghi ra file để bạn review và approve.

📐 Format Plan chuẩn
Plan luôn bao gồm: Goal (mục tiêu), Assumptions (giả định), Plan Steps (2–10 phút mỗi bước), Verification cho từng bước, Risks & Mitigations, và Rollback Plan.

Cách sử dụng

chat command 
# Gõ trong chat Antigravity:
/superpowers-write-plan Implement JWT authentication với refresh tokens.
Stack: FastAPI, SQLAlchemy, PostgreSQL. Rate limit 5 req/min.

AI sẽ:

  • Phân tích yêu cầu và liệt kê giả định
  • Tạo plan từng bước nhỏ (2–10 phút/bước), mỗi bước có verification command
  • Ghi plan ra artifacts/superpowers/plan.md
  • Chờ bạn approve trước khi bắt đầu thực thi
⚠️ Quan trọng
Đây là lúc dễ phát hiện sai hướng nhất — review kỹ plan trước khi approve. Sau khi AI đã execute, việc thay đổi hướng sẽ tốn nhiều công hơn.

Output mẫu — plan.md

artifacts/superpowers/plan.md 
## Goal
Implement JWT auth với refresh tokens cho FastAPI app

## Assumptions
- PostgreSQL đang chạy ở localhost:5432
- SQLAlchemy 2.x đã cài
- bcrypt rounds=12

## Plan Steps
Step 1: Tạo User model + migration [5 phút]
  Files: models/user.py, alembic/versions/001_add_users.py
  Verify: alembic upgrade head → success

Step 2: Implement AuthService [8 phút]
  Files: services/auth.py
  Verify: pytest tests/test_auth_service.py → pass

Step 3: Tạo /auth/register + /auth/login endpoints [6 phút]
  Files: routers/auth.py
  Verify: pytest tests/test_auth_routes.py → pass

## Rollback Plan
alembic downgrade -1 để revert migration nếu có vấn đề
Workflow 03

⚙️ /superpowers-execute-plan

Thực thi plan từng bước, có verification sau mỗi bước. AI không bỏ qua bước nào, không chạy ẩu.

1
Đọc plan.md
AI đọc artifacts/superpowers/plan.md, xác nhận hiểu đúng từng bước trước khi bắt đầu.
2
Execute từng step
AI thực hiện từng bước nhỏ trong plan — viết code, tạo file, chạy migration...
3
Verify sau mỗi bước
Chạy verification command đã định trong plan. Nếu fail → dừng lại và báo ngay, không tiếp tục.
4
Ghi artifacts
Code diff, test results, logs được ghi ra artifacts/superpowers/ để bạn review.
💡 Tip
Bạn có thể thêm yêu cầu khi gọi execute: /superpowers-execute-plan Thêm logout endpoint nữa nhé. — AI sẽ cập nhật plan trước khi thực thi.
Workflow 04 · Advanced

⚡ /superpowers-execute-plan-parallel

Tính năng nâng cao — chạy các bước plan độc lập song song bằng nhiều subagents, tiết kiệm đáng kể thời gian.

🔀 Khi nào dùng Parallel?
Dùng khi plan có nhiều task không phụ thuộc nhau, ví dụ: "thêm 3 API endpoints độc lập", "implement 3 modules riêng biệt", "viết tests cho 4 services khác nhau".
Sequential (bình thường)
Feature A → Feature B → Feature C

Tổng: 45 phút
Agent: 1 × 3 tasks
Parallel (execute-parallel)
Feature A ↗
Feature B → (parallel)
Feature C ↘

Tổng: ~15 phút
Agents: 3 × 1 task

Log subagents

Mỗi subagent ghi log riêng vào artifacts/superpowers/subagents/. Bạn có thể xem chi tiết từng agent đã làm gì:

bash 
# Xem log từng subagent
ls artifacts/superpowers/subagents/
# Output:
subagent-1-feature-auth.log
subagent-2-feature-payment.log
subagent-3-feature-notification.log
Workflows 05-08

🛠 Các Workflow còn lại

/superpowers-review
Review code staged changes hoặc toàn bộ project. Tìm bugs, security holes, code smells, thiếu test. Output là report với severity levels (critical/warning/suggestion).
Dùng trước commit: /superpowers-review
/superpowers-debug
Debug có hệ thống: reproduce bug → tìm root cause → fix tối thiểu → viết regression test → verify. Không fix ẩu, luôn có test chứng minh bug đã được sửa.
Kèm error: /superpowers-debug [paste stack trace]
/superpowers-finish
Workflow hoàn thiện task: remove debug code, cập nhật README, check TODOs, final test run, viết commit message chuẩn conventional commits.
Khi task xong: /superpowers-finish
/superpowers-reload util
Reload toàn bộ framework (rules + workflows + skills) cho session hiện tại. Dùng khi AI bắt đầu không tuân thủ rules hoặc sau khi bạn cập nhật files trong .agent/.
Fix AI quên rules: /superpowers-reload
⚠️ Nếu AI không tuân workflow
Prompt vào chat: "Please follow the superpowers-execute-plan workflow and write artifacts to disk." — hoặc chạy /superpowers-reload để AI đọc lại toàn bộ rules.

Skills · Nâng cao

✏️ Tạo Skill tuỳ chỉnh

Bạn có thể tạo skill riêng cho project, tech stack, hoặc quy trình của team. Skill là một thư mục trong .agent/skills/:

.agent/skills/my-custom-skill/
├── skill.md← Mô tả skill và khi nào dùng
└── instructions.md← Hướng dẫn chi tiết cho AI

Ví dụ — Skill cho Next.js + Prisma

.agent/skills/nextjs-prisma/skill.md 
# Skill: nextjs-prisma

## Khi nào dùng
Áp dụng khi làm việc với Next.js App Router + Prisma ORM.

## Stack cụ thể
- Next.js 14+ App Router
- Prisma với PostgreSQL
- TypeScript strict mode
- shadcn/ui components

## Rules bắt buộc
1. Luôn dùng Server Actions cho mutations, không dùng API routes thừa
2. Prisma queries phải có error handling với try/catch
3. Schema changes phải có migration, không dùng db push
4. Components phải có loading.tsx và error.tsx

## Verification commands
- npx prisma migrate dev → phải thành công
- npx tsc --noEmit → không có type errors
- npm run build → phải build được
✅ Best Practice
Tạo một skill riêng cho mỗi tech stack chính của team. Ví dụ: react-native-skill, django-skill, terraform-skill. AI sẽ tự động áp dụng đúng conventions mà không cần nhắc đi nhắc lại.

✍️ Viết yêu cầu rõ ràng

❌ Mơ hồ
"Build authentication cho app của tôi"

AI không biết: stack gì? JWT hay session? Có refresh token không? Database nào?
✅ Cụ thể
"Implement JWT auth với refresh tokens. Stack: FastAPI + PostgreSQL + SQLAlchemy. Rate limit login 5 req/min. Access token 15m, refresh 7d."

📁 Tổ chức artifacts

  • Commit artifacts vào git để team có thể review plan và quyết định trước khi code
  • Comment vào plan.md để AI đọc feedback và điều chỉnh approach
  • Kiểm tra subagents/ logs khi dùng parallel để verify từng agent đã làm đúng
  • Xóa artifacts/ cũ định kỳ để tránh AI đọc nhầm context cũ

🔄 Reload đúng lúc

💡 Khi nào cần /reload
Antigravity có thể cache file cũ. Sau khi bạn sửa bất kỳ file nào trong .agent/, hoặc khi AI bắt đầu không tuân thủ rules, gọi /superpowers-reload để buộc AI đọc lại toàn bộ framework.
Troubleshooting

🔧 Xử lý sự cố thường gặp

ERROR Antigravity không nhận diện các workflows khi gõ slash command ▶

Nguyên nhân: Antigravity chưa được mở đúng folder chứa .agent/, hoặc đang cache cấu hình cũ.

Fix:

  • Dùng File → Open Folder và chọn đúng thư mục project (chứa .agent/)
  • Gõ /superpowers-reload để force reload
  • Restart Antigravity nếu vẫn không nhận
ERROR Gemini CLI không tìm thấy hoặc lỗi permission ▶

Nguyên nhân: Gemini CLI chưa cài, hoặc không có trong PATH.

Fix:

bash
# Kiểm tra gemini CLI
gemini --version

# Nếu không có, cài qua npm
npm install -g @google/generative-ai

# Hoặc cài qua pip
pip install google-generativeai
WARN AI không follow Superpowers rules, bắt đầu code ngay không lên plan ▶

Nguyên nhân: AI cached session cũ và không đọc lại rules.

Fix: Gõ một trong hai:

chat command
/superpowers-reload

# Hoặc prompt thủ công:
Please follow the superpowers-execute-plan workflow
and write artifacts to disk.
WARN Artifacts không được ghi ra disk (artifacts/ trống) ▶

Nguyên nhân: AI đang chạy trong chế độ không có quyền ghi file, hoặc quên quy tắc ghi artifacts.

Fix:

  • Kiểm tra Antigravity có quyền ghi vào thư mục project
  • Gõ: "Please write all artifacts to artifacts/superpowers/ as required by the rules."
  • Tạo thư mục artifacts/superpowers/ thủ công rồi thử lại
INFO Sau khi cập nhật .agent/ files, AI vẫn dùng version cũ ▶

Nguyên nhân: Antigravity cached files cũ trong session.

Fix: Gõ /superpowers-reload — đây sẽ force Antigravity đọc lại tất cả files trong .agent/ cho session hiện tại.

✅ Kiểm tra health tổng thể
Chạy lệnh này trong chat để verify mọi thứ đang hoạt động đúng:

/superpowers-reload

AI sẽ liệt kê: số workflows loaded, số skills loaded, và xác nhận sẽ follow rules cho phần còn lại của session.
