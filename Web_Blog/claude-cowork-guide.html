SETUP

Cài đặt & Yêu cầu hệ thống

Những gì bạn cần trước khi bắt đầu

Yêu cầu tối thiểu
🍎 macOS
Mac (Apple Silicon hoặc Intel)
macOS Sonoma (14+) trở lên
RAM khuyến nghị: 8GB trở lên
Dung lượng trống: ~2GB (cho VM download lần đầu)
Computer Use: yêu cầu Apple Silicon (M1/M2/M3/M4)
🪟 Windows
Windows 10/11 (64-bit)
Ra mắt Windows: 10/02/2026 — đầy đủ tính năng
WSL2 được cài sẵn (Cowork tự cấu hình)
RAM khuyến nghị: 8GB trở lên
Full parity: tất cả tính năng như macOS
Các bước cài đặt từng bước
1
Tải Claude Desktop App
Vào claude.com/download → chọn macOS hoặc Windows → tải về và cài đặt. App luôn accessible từ dock/taskbar.
2
Đăng ký gói trả phí (Pro / Max / Team)
Cowork chỉ có trên paid plans: Pro ($20/tháng), Max ($100-200/tháng), Team ($30/user), Enterprise. Pro đủ dùng để bắt đầu; Max có usage limit cao hơn.
3
Mở Desktop App → Tìm mode selector
Trong giao diện Desktop app, bạn thấy thanh chọn chế độ gồm Chat | Cowork | Code. Click vào tab Cowork.
4
Lần đầu: Download VM (~2GB)
Cowork chạy trong một isolated virtual machine trên máy bạn. Lần đầu tiên sẽ tải xuống ~2GB. Mất khoảng 5–10 phút. Các lần sau khởi động nhanh hơn nhiều.
5
Cấp quyền thư mục (Folder Permissions)
Cowork hỏi bạn muốn cho phép truy cập thư mục nào. Ví dụ: ~/Documents, ~/Downloads. Chỉ những thư mục bạn cho phép mới accessible. Bạn kiểm soát hoàn toàn.
6
Kết nối Connectors (tuỳ chọn)
Vào Settings → Extensions để kết nối Gmail, Slack, GitHub, Notion, Google Drive, v.v. Mỗi connector cần xác thực OAuth một lần. Sau đó Claude có thể đọc/ghi dữ liệu từ các dịch vụ đó.
7
Giao task đầu tiên!
Gõ mô tả task trong Cowork tab. Claude hiển thị kế hoạch thực hiện → bạn duyệt → Claude bắt đầu làm việc. Desktop app phải để mở trong khi Claude đang chạy task.
⚠️
Lưu ý quan trọng: Desktop app phải ở trạng thái mở trong khi Claude đang thực hiện task. Nếu bạn đóng app hoặc máy tính ngủ, session sẽ kết thúc. Với Dispatch (task theo lịch), bạn vẫn cần máy tính đang chạy.
TASKS

Tasks Mode — Giao việc cho Claude

Cách Claude lập kế hoạch và thực hiện task nhiều bước

Quy trình hoạt động của một Task
①
Bạn mô tả task bằng ngôn ngữ tự nhiên
Ví dụ: "Đọc tất cả file PDF trong thư mục ~/Downloads/Reports, tóm tắt từng file, gom vào một document Word và lưu vào ~/Documents/Summary.docx"
②
Claude lập kế hoạch và hiển thị cho bạn
Claude chia nhỏ task thành subtasks, liệt kê từng bước, cho bạn thấy sẽ dùng tool/connector nào. Bạn có thể sửa, bổ sung, hoặc approve trước khi bắt đầu.
③
Claude thực hiện — bạn theo dõi real-time
Progress indicators hiện từng bước đang chạy. Claude cho bạn thấy reasoning. Bạn có thể can thiệp, chỉnh hướng hoặc để tự chạy hết.
④
Claude dừng lại khi cần quyết định quan trọng
Nếu sắp làm điều gì có tác động lớn (gửi email, xóa file, tạo PR...), Claude hỏi xác nhận trước. Không bao giờ tự làm tức thì mà không có approval.
⑤
Kết quả: Deliverable hoàn chỉnh
Claude trả về sản phẩm cuối cùng — file Word, spreadsheet, email draft, code file... Không phải hướng dẫn từng bước — là kết quả thực sự.

DISPATCH

Dispatch — Task chạy theo lịch & từ mobile

Giao task từ điện thoại, hoặc lên lịch tự động — Claude làm khi bạn vắng

📡
DISPATCH
Giao việc từ điện thoại, Claude làm trên laptop
Dispatch cho phép bạn assign task cho Cowork từ bất kỳ đâu — qua mobile app, hoặc cài sẵn lịch lặp lại. Claude làm việc khi bạn đang nghỉ, và bạn về thấy kết quả đã sẵn sàng.
Hai cách dùng Dispatch
📱 REMOTE DISPATCH
Giao task từ điện thoại
Cài Claude app trên iPhone/Android → kết nối với Desktop Cowork

Nhắn task trên điện thoại → Claude nhận và bắt đầu làm trên laptop ngay lập tức.

Ví dụ thực tế: Đang trong cuộc họp, nhớ ra cần prep deck cho meeting chiều → mở Claude mobile → nhắn "Tạo slide từ file notes trong ~/Notes/Q4/" → về đến bàn thấy deck đã xong.
⏰ SCHEDULED DISPATCH
Task lặp lại theo lịch
Gõ schedule trước mô tả task → chọn tần suất: daily / weekly / custom cron.

Claude tự chạy đúng giờ, không cần bạn ở đó.

Ví dụ thực tế: Mỗi sáng 7h, Claude quét email mới → tóm tắt → tạo daily briefing → lưu vào Notion. Bạn mở máy thấy briefing đã sẵn sàng.

COMPUTER USE

Computer Use — Claude điều khiển màn hình

Claude thấy màn hình bạn, di chuyển chuột, gõ bàn phím — làm được mọi thứ bạn làm

🖥️
COMPUTER USE
Mắt và tay trên desktop của bạn
Computer Use cho phép Claude nhìn màn hình, click chuột, gõ phím, mở ứng dụng — giống như có người thật đang điều khiển máy tính của bạn. Mọi thứ bạn làm được trên GUI đều làm được.
Computer Use có thể làm gì?
🖱️ GUI AUTOMATION
Điều khiển ứng dụng desktop
Mở bất kỳ app nào (Figma, Excel, Photoshop...)
Click vào menu, button, form, dropdown
Drag & drop file, scroll, zoom
Ví dụ: "Mở Figma, xuất tất cả artboard thành PNG, lưu vào ~/exports/"
🌐 BROWSER AUTOMATION
Duyệt web, điền form
Kết hợp với Claude in Chrome: navigate URL, click link, điền form, scrape data
Ví dụ: "Vào portal HR, download payslip 3 tháng gần nhất, lưu vào ~/Documents/Payslips/"
📋 LEGACY APP HANDLING
App cũ không có API
Nhiều phần mềm nội bộ công ty không có API. Computer Use giải quyết bằng cách thao tác GUI trực tiếp.
Ví dụ: "Vào phần mềm kế toán ABC, export báo cáo Q4, lưu thành Excel"
🔗 CROSS-APP WORKFLOW
Nối nhiều app với nhau
Copy data từ app này sang app khác mà không cần API integration.
Ví dụ: "Đọc data từ Excel, nhập vào CRM, sau đó tạo invoice trong phần mềm kế toán"
⚠️
Yêu cầu: Computer Use yêu cầu Pro hoặc Max plan và hiện chỉ dùng được trên macOS. Tính năng này vẫn đang trong research preview. Claude sẽ hỏi xác nhận trước khi thực hiện bất kỳ thao tác GUI nào.
FILE ACCESS

File Access — Truy cập file local trực tiếp

Không cần upload thủ công — Claude đọc/ghi thẳng vào thư mục bạn cho phép

Cách File Access hoạt động

Khi cài Cowork, bạn chỉ định các thư mục được phép truy cập. Claude hoạt động trong isolated VM nhưng có mount point vào các thư mục đó. Kết quả là Claude có thể:

  • Đọc bất kỳ file nào trong thư mục được phép: PDF, Word, Excel, markdown, JSON, code...
  • Tạo file mới và lưu trực tiếp vào thư mục
  • Sửa file hiện có (với confirmation của bạn)
  • Di chuyển, đổi tên file theo quy tắc bạn đặt ra
Ví dụ task dùng File Access FILE 
# Task 1: Tóm tắt nhiều file PDF
"Đọc tất cả PDF trong ~/Downloads/Papers/,
 tóm tắt từng bài thành 3 bullet points,
 gom vào ~/Notes/paper-summaries.md"

# Task 2: Reorganize file theo ngày
"Đi qua ~/Downloads/, tìm tất cả file có tên
 chứa 'invoice', đổi tên thành YYYY-MM-DD_invoice_*.pdf
 và di chuyển vào ~/Documents/Invoices/[year]/"

# Task 3: Phân tích code base
"Đọc toàn bộ src/ folder trong project của tôi,
 phân tích dependencies, tìm dead code,
 tạo báo cáo markdown về kỹ thuật nợ (tech debt)"
🔒
Bảo mật: Claude chỉ truy cập được các thư mục bạn chủ động cho phép. Bạn có thể thay đổi permissions bất cứ lúc nào trong Settings → Folder Access. Mọi thứ chạy local — không có data nào gửi lên cloud ngoài prompt và response với model.

MCP

MCP Servers — Giao thức kết nối đa năng

Model Context Protocol — viết một lần, dùng được với mọi MCP client

MCP là gì? (Giải thích đơn giản)

MCP (Model Context Protocol) là tiêu chuẩn mở do Anthropic phát triển, cho phép bất kỳ phần mềm nào "nói chuyện" với Claude theo cách chuẩn hóa. Giống như USB — bạn cắm một cái gì đó tuân theo chuẩn USB vào, là nó hoạt động ngay.

Với MCP, developer viết integration một lần → bất kỳ MCP-compliant client nào (Claude Desktop, Cowork, Claude Code, Cursor...) đều dùng được.

📖
Resources
Data read-only mà model có thể lấy: nội dung file, rows DB, API responses. Claude "đọc" context từ đây.
⚡
Tools
Actions Claude có thể gọi: tạo issue, gửi tin nhắn, chạy query, gọi API. Claude "làm" việc qua đây.
📝
Prompts
Template prompt định sẵn cho các tác vụ phổ biến. Claude sử dụng để thực hiện task nhất quán.

  "mcpServers": {
    "github": {
      "command": "npx",

      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    },
    "notion": {
      "command": "npx",
      "args": ["-y", "@notion/mcp-server"],
      "env": {
        "NOTION_API_KEY": "secret_xxxxxxxxxx"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://user:pass@localhost:5432/mydb"]
    }
  }
}

Best practice bảo mật MCP: Luôn cấp quyền tối thiểu cần thiết cho mỗi server. Ví dụ: GitHub token chỉ cần quyền read-only nếu chỉ để review PR. Nếu server bị compromise, blast radius sẽ bị giới hạn. Track MCP config trong version control như dotfiles.

SECURITY

Bảo mật & Giới hạn cần biết

Những điều quan trọng để dùng Cowork an toàn và hiệu quả

Mô hình bảo mật của Cowork
✅ BẢO MẬT TỐT
Isolated VM — không thoát ra ngoài
Claude chạy trong virtual machine cô lập — không thể tự ý truy cập file ngoài thư mục được phép, không thể cài phần mềm ngoài VM, không thể can thiệp vào OS.

Mọi action quan trọng đều cần xác nhận của bạn trước khi thực hiện.
⚠️ RỦI RO CẦN ĐỂ Ý
Prompt Injection & Scheduled Tasks
Nếu Claude đọc file độc hại chứa "instruction" ẩn, có thể bị lợi dụng (prompt injection). Khi dùng Dispatch, task chạy khi bạn không ở đó — thiết lập cẩn thận và giới hạn scope.

Monitor scheduled task creation như tín hiệu bảo mật quan trọng.
Giới hạn thực tế cần biết
  • Máy phải mở: Cowork không chạy khi laptop ngủ. Dispatch cũng không chạy được nếu máy tắt/sleep.
  • Usage limits: Pro users hit limit sớm hơn Max. Task phức tạp nhiều steps tốn nhiều usage hơn.
  • Rate limits & sub-agents: Khi task spawn nhiều sub-agents, rate limit cộng dồn nhanh hơn.
  • Computer Use preview: Tính năng này đang nghiên cứu, chỉ macOS, có thể không ổn định.
  • Không thay thế domain expertise: Cowork làm tốt khi bạn biết muốn gì — nó không tự "biết" business logic đặc thù của công ty bạn.
⚠️
Quy tắc vàng: Không cấp quyền Cowork truy cập thư mục nhạy cảm (~/ssh, secret keys, password manager export). Dùng nguyên tắc least privilege — chỉ cho phép những gì thực sự cần. Với MCP servers, luôn dùng API key có scope tối thiểu.
